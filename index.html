AI 에이전트 & 하네스 엔지니어링 — 연구·방법론

멀티에이전트 오케스트레이션 실패 분석
멀티에이전트가 왜 무너지는가를 정량화 — 맥락 붕괴 41.8%·유령 위임 36.9%·검증 오류 21.3%. 단일 오케스트레이터·Blackboard·도메인 독립의 3원칙으로 현실적 L3 설계 도출
에이전트 하네스 엔지니어링
"프롬프트가 아니라 루프를 설계한다." 안쪽 루프(ReAct)와 바깥 루프(탐색·검증·기록)의 이중 구조로 장기 실행 에이전트를 설계하는 방법론
Fable-5 Traces 분석 (공개 연구)
단명한 프런티어 모델의 실제 코딩 궤적 4,665행을 분석 → 검증 내장 루프의 86.7%가 10스텝 내 완료됨을 발견. 인터랙티브 뷰어·덱·인포그래픽으로 공개 (Vercel + GitHub)
GraphRAG 신뢰형 QA
Neo4j + LangChain. 불안정한 단발 Cypher 생성을 의도분류·엔티티링킹·고정 템플릿으로 대체해 구문오류 0·일관 정답 달성
병렬 컨텍스트 압축 + Merge Verifier
긴 세션을 블록 단위로 분해 → 5필드 스키마로 압축 → verifier의 4검사(coverage·contradiction·evidence·replay)로 핸드오프 손실을 선검출
LLM + ML 앙상블 설계
Claude(Sonnet) + XGBoost의 piecewise 가중 평균, Batch API + prompt caching으로 정확도·비용을 최적화한 예측 시스템 설계
